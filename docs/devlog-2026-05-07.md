## 1. Cykling end-to-end

### Race-agnostiske jerseys (commit `a7f89c3`)
**Problem**: Internt brugte vi `yellow/green/polka/white` som jersey-keys,
men farverne varierer per race (Giro = rosa, Vuelta = rød osv).

**Fix**:
- Renamede DB-værdier: `yellow → leader`, `green → points`, `polka → mountain`, `white → youth`
- Scraper (`sync_results.py::_parse_jerseys`) emitter nye keys
- Scoring (`lib/calculateCyclingPoints.ts::JERSEY_POINTS`) opdateret
- UI viser race-agnostiske badges (FØR/PT/BT/UT) med traditionelle farver

### GC-multiplier (commit `a7f89c3`)
Top-10 sammenlagt får placerings-bonus per etape:
- GC 1: ×1.4 · GC 2-3: ×1.3 · GC 4-5: ×1.2 · GC 6-10: ×1.1

**Tekniske ændringer**:
- `cycling_results.gc_position_after` (int) — top-20 GC scrapes per etape
- `cycling_scores.gc_multiplier` (numeric) — generated column `total_points`
  inkluderer nu `× gc_multiplier`
- Scraper henter GC fra PCS classification table per stage

### Hviledag-transfers (commit `a7f89c3`)
GT-spillere kan swappe op til 3 ryttere per hviledag.

**Tekniske ændringer**:
- `cycling_races.rest_days date[]` — rest day datoer per race
- `cycling_squad_transfers` — log af swaps med snapshot-kategori
- `lib/cyclingTransfers.ts::getEffectiveSquadRiders()` — beregner brutto-trup
  ved en given dato (anvender historiske transfers)
- Bench-penalty bruger nu effektiv trup, ikke original
- API: `POST /api/games/[id]/cycling/transfer` (host validation, max 3 swaps,
  startlist-only, samme cat/team-grænser)
- UI: `components/cycling/TransferModal.tsx` + knap i [LineupBuilder.tsx](../components/cycling/LineupBuilder.tsx)

### Cycling block winners + auto-flip (commit `138ea46`)
**Problem**: `cycling_blocks` havde ingen `status`-kolonne i DB → leaderboard
filtrerede på et felt der altid var undefined → ingen B. SEJR.

**Fix**:
- Migration: `ALTER TABLE cycling_blocks ADD COLUMN status text DEFAULT 'active'`
- `lib/calculateCyclingPoints.ts::updateCyclingBlockStatuses()` — flipper blokke
  til `finished` når alle linkede races er finished
- Kaldes som del af point-beregnings-pipeline

### Profile-baseret bonus (commits `670b296`, `5f36c1b`)
- Etape-info viser hvilke roller der får boost (e.g. mountain → Grimpeur ×1.8)
- Bunden af gameroom: kompakt RolesGuide tabel + link til [/games/cycling-guide](../app/games/cycling-guide/page.tsx)

## 2. Live-data arkitektur (4-fase refactor)

**Før**: 3 konkurrerende mekanismer
- Server-render leaderboard ved sideload
- Client-poll `/api/games/[id]/live-matches` hvert 30s
- `router.refresh()` trigget ved match-finish
- Cron `/calculate-points` hver 10. min

**Efter**: én mekanisme
- `/api/games/[id]/state` returnerer matches + leaderboard + block-standings
- `useGameState(gameId)` poller adaptive: 30s ved aktive kampe, 2 min ved idle
- Point-beregning trigges direkte fra `syncMatchScores` ved finished-flip
- Cron som safety-net (hvert 30. min)

### Nøgle-filer
- [lib/gameState.ts](../lib/gameState.ts) — `getGameMatches`, `getGameLeaderboard`,
  `getGameState`, `getActiveBlockStandings`

  + `useGameStateContext` (+ optional)
- [components/games/FootballLiveSection.tsx](../components/games/FootballLiveSection.tsx) —
  wrapper der renderer `Leaderboard` + `BlockLeaderboard` med live data

### Faser (committet separat for rollback-mulighed)

2. **Fase 2** (`ca93625`) — `lib/gameState.ts` + `/state` endpoint + hook

## 3. Gameroom lifecycle

### Auto-arkivering (commit `06caa89`)
Cycling gamerooms arkiveres automatisk 14 dage efter sidste løb:
- Daglig cron `/cycling-archive-check` (08:00 UTC)
- 7-dages push-varsel (host får CTA-tekst, øvrige får info)
- 14-dages auto-flip til `status='finished'`
- Vises automatisk i dashboardets "Afsluttede spil"-sektion

DB: `games.archive_warning_sent_at timestamptz`

### Tilføj løb (commit `04390c1`)
Host kan appende nye løb til eksisterende/arkiveret gameroom:
- `POST /api/games/[id]/cycling/add-races` (host-only)
- Reaktiverer arkiverede spilrum (`status='finished' → 'active'`)
- `generateCyclingBlocks()` er nu appendable — skipper løb med eksisterende blok
- Page: [/games/[id]/cycling/add-races](../app/games/[id]/cycling/add-races/page.tsx)
- Form: [components/cycling/AddRacesForm.tsx](../components/cycling/AddRacesForm.tsx)
  (afviser allerede-færdige løb)

### Arkiveret-banner + ticker (commits `69e570c`, `d483e09`)
- Banner ved toppen af `finished` gamerooms (host får +CTA-knap)
- Newsbar viser ticker-beskeder ved blok-livscyklus:

## 4. Cykling forfining

### Squad-grænser (commit `f8a8e51`)
**Problem**: Standard-regler (max 3+5+5+5+7=25) antager Tour-niveau startliste.
Mindre løb som Romandiet har færre topryttere → umuligt at fylde 25.

**Fix**: [lib/cyclingSquadLimits.ts](../lib/cyclingSquadLimits.ts) beregner
reelle grænser per blok:
```
limit[cat] = min(default[cat], avail_in_block_startlists[cat])
maxTotal   = min(25, sum(limit[1..5]))
```
Brugt i POST endpoint + squad page + SquadBuilder. Banner vises hvis
grænserne er reduceret.

### Block ordering (commit `a5daa55`)
Stage races får nu kronologisk `block_order` baseret på `start_date` (ikke
race-PK rækkefølge fra DB).

### Profile-derivering (commit `18ef0ef`)
PCS skiftede semantik på deres `<span class="profile pX">` CSS-klasser
(p5 betyder nu mountain, var ITT). Skifter til heuristik baseret på
`profile_score`:
- `< 25` → flat · `25-99` → hilly · `≥ 100` → mountain

Anvendt i `sync_race_info.py` + one-shot opdatering af 133 stages.

### Ghost-rytter markering (commit `43e4855`)
Hvis en rytter i brutto-trup ikke er på blokkens startliste:
- Rød "IKKE STARTER"-badge i SquadBuilder
- Banner: "⚠️ N ryttere er ikke på startlisten og scorer 0 point"

### Replace-all startlist sync (commit `8cffc77`)
**Problem**: `sync_results.py` upsertede startlister men slettede aldrig
ryttere der røg af → ophobning af stale entries (Giro havde 202 i DB
mens PCS rapporterede 184).

**Fix**: Sletter alle eksisterende rows for racen før insert.

### Andre fixes
- `7db17db` — default til aktiv (ikke-finished) blok + "Prolog" i stedet for "Etape 0"
- `1e2c220` — vis blok-vinder 🏆 + brugernavn i tab
- `e7727f5` — B. PT viser nu kun nuværende bloks point (ikke sæson-total)
- `d284a52` — allerede-færdige løb er ikke-valgbare i "Tilføj løb"

## Tilbagevendende mønstre / arkitektur-noter

### Snapshot-pattern (kategori, transfer-info)
- `cycling_squad_riders.category_slot` — fastlåst ved squad-udtagelse
- `cycling_squad_transfers.rider_in_category` — fastlåst ved swap
- Forhindrer UCI-ranking-drift midt i blok

### Fail-closed validering
- Lineup-deadline: hvis `start_date` mangler → afvis (i stedet for at lade
  igennem). Manglende data = sikker default.

### Idempotente DB-operationer
- `generateCyclingBlocks` skipper løb der allerede har `cycling_block_id`
- `updateCyclingBlockStatuses` kun finished-flip, aldrig reverse
- Migrations bruger `IF NOT EXISTS` / `IF EXISTS` clauses

### Replace-all > upsert for tidsfølsomme lister
- Startlister: slet alle for race før insert (forhindrer stale entries)
- Bet-distribution: omberegnet hver gang (genereret fra bet-tabellen)
