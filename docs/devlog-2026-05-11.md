for brugere, bare ikke obligatorisk for admin endnu.

**Kendt issue (lГёst i `da07a8c`)**: MFA-status hang efter password-sign-in
hvis brugeren ikke havde TOTP вҖ” fix var at refreshe session efter login.

### Audit-log pГҘ admin (commit `728480a`)

- Helper i [lib/auditLog.ts](../lib/auditLog.ts) вҖ” kaldes fra alle admin-mutationer

### Resend email integration (commits `022c10e`, `3aa3bfa`)

- Templates til welcome + password-reset + bet-deadline-reminder
- Supabase auth-emails konfigureret separat вҖ” se [docs/supabase-email-templates.md](supabase-email-templates.md)
- Klar til at fyre op nГҘr API-key er sat i prod env

### Logout-robusthed (commits `0a0fd84`, `813f15b`)
**Problem**: Client-side `supabase.auth.signOut()` fejlede hvis sessionen var
i AAL2-state вҖ” brugeren sad fast.

**Fix**: Dedikeret `/logout` server-route der clearer cookies + redirect'er.
Fail-safe button kalder den uanset AAL-status.

## 4. Cykling вҖ” efter-alfa fixes

### Auto-sync i Railway-cron (commit `233dddb`)
Stage-resultater synces nu automatisk via Railway cron вҖ” tidligere skulle
det trigges manuelt. [railway/index.ts](../railway/index.ts) kГёrer pГҘ `*/15 * * * *` i kursus-vinduer.

### Sync-stabilisering (commits `1b87cd1`, `62209e1`, `8eabfe1`)
- **Backoff**: drop unГёdig points-recalc-storm nГҘr der ingen ГҰndringer er
- **Sanity-check**: skip fake-results-tabel (PCS poster lejlighedsvis
  placeholder-tabel fГёr den rigtige er klar)
- **Skip fremtidige stages**: scraperen forsГёgte ellers at parse stages der
  ikke er kГёrt endnu

### DNF-hГҘndtering (commit `1c9298d`)
- Ryttere markeret DNF disable'es i lineup-picker
- "UD AF LГҳBET" badge vises pГҘ rytter-kort
- Eksisterende lineups med DNF-ryttere fГҘr 0 point for de pГҘgГҰldende ryttere
  (ikke straf, bare ingen score)

### Race vs stage status audit (commits `45ed66b`, `bdba3a3`, `9936662`, `1347e71`, `344ec7f`)
**Problem**: Flere steder blev `race.status` brugt som proxy for `stage.status`
вҖ” men en race kan vГҰre `active` mens en specifik stage er `finished` eller
omvendt. Resulterede i forkert live-badge og resultater der ikke viste.

**Fix**:
- Full audit af alle steder hvor race-status blev tolket som stage-status
- Live-badge nu per stage, ikke per race
- Lineup-API viser resultater per stage uden status-filter
- Default-tab springer ikke lГҰngere over dagens etape (+ advarsel hvis ingen)
- Migration: unique constraint pГҘ `cycling_lineups (squad_id, stage_id)`

### Hydration mismatch fix (commit `798c13b`)
`formatCyclingDeadline()` brugte `new Date()` direkte вҶ’ mismatch mellem
server (UTC) og client (Europe/Copenhagen). Fix: alle date-formatters
tager nu eksplicit `timeZone: 'Europe/Copenhagen'`. OgsГҘ `activeTab`
default'er nu deterministisk.

### Mobile-layout for all-lineups (commit `4de0c84`)
Nikolaj's iPhone 14 Pro Max viste navne afskГҘret. Fix: 1-kolonne layout
pГҘ mobile + korte rolle-labels (`G/K/S/J` i stedet for fulde navne).
